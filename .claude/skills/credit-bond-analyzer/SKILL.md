# Skill: credit-bond-analyzer

## 输入

- `issuer`: 主体名称
- `bond_code`: 债券代码，可选
- `mode`: `issuer_review` / `bond_admission` / `spread_weekly` / `warning_scan`

## 输出

- `output/credit/{issuer}_{mode}.md`
- `output/credit/{issuer}_{mode}.html`

## 步骤

### 步骤 1：材料与数据清单

- [ ] 主体基本信息、股权结构、行业分类
- [ ] 财务三表和关键科目：现金、短债、有息债务、经营现金流、资本开支
- [ ] 债券余额、期限分布、担保和特殊条款
- [ ] 外部支持：政府/股东/集团/银行授信
- [ ] 利差：等级利差、行业利差、期限利差、个券利差
- [ ] 市场：成交活跃度、估值偏离、质押便利性

### 步骤 2：调 Subagent

`cn-credit-issuer-reader` 抽取主体、财务、外部支持、债项条款、利差和预警字段。

### 步骤 3：五层信用框架

1. 宏观信用环境：宽信用/紧信用、再融资窗口、监管约束
2. 行业景气与政策：城投、产业、地产、金融分别判断
3. 主体资质：业务稳定性、现金流、债务结构、再融资
4. 外部支持：支持能力和支持意愿分开写
5. 债项与市场：条款、担保、期限、流动性、利差补偿

### 步骤 4：两个结论分开

- 能不能买：是否在可投资范围，核心尾部风险是什么
- 值不值得买：当前利差是否补偿风险和流动性

### 步骤 5：投后跟踪

- 价格异常、利差跳升、成交枯竭、评级/舆情/公告
- 银行授信变化、债券发行失败、同区域/同行业风险扩散

## 检查清单

- [ ] 没有把评级当结论
- [ ] 没有把主体背景或担保当充分安全垫
- [ ] 高票息先解释风险补偿来源
- [ ] 明确区分风险结论和价格结论
